**分析**: 过滤器被正确加载，但trade log完全一致。可能原因：
- 2024年BEAR期主要在年中调整期，持续时间短，filter-confirm-days=2使部分BEAR信号未被确认

## ⚠️ 核心发现：过滤器为何在多数窗口无效？

### 1. NEUTRAL regime 占比过高
- 6个窗口中5个的NEUTRAL占比>50%
- NEUTRAL时 `position_limit=1.0`，等于无限制
- **过滤器实际上只在BEAR regime下才有限制作用**

### 2. filter-confirm-days=2 减少了有效BEAR天数
- 需要连续2天确认才能切换到BEAR
- 短暂反弹打断BEAR确认
- 建议：降低confirm-days或增加BEAR确认天数

### 3. 最大回撤往往发生在仓位最大的时期
- 策略在趋势初期全仓买入
- 止损触发后才减仓
- position_limit只能控制新买入量，无法改变已有持仓的亏损

### 4. position_size vs position_limit 的双重上限
```
per_stock_cash = min(avail_cash/fill_count, cash*position_size)
                = min(cash*0.4/5, cash*0.2)
                = min(80,000, 200,000)  ← BEAR regime
                = 80,000 (正确)
```
逻辑本身正确，但限制的是"每只股票分配金额"，而非"总亏损"。

## 📋 结论与建议

### 过滤器有效性评估

| 场景 | 有效性 | 原因 |
|------|--------|------|
| 高BEAR比例(>25%)，长下跌期 | ✅ 有效 | Window F: +1.23%收益，-1.03%MaxDD |
| 中BEAR比例(15-25%) | ⚠️ 有限 | Window B: 部分仓位变化，但MaxDD不变 |
| 低BEAR比例(<10%) | ❌ 无效 | Window C: BEAR天数太少 |
| 短期急跌后快速反弹 | ❌ 无效 | Window D: 止损先触发 |

### 建议改进方向

1. **调整 position_limit 映射**
   - 当前 BEAR→0.4, NEUTRAL→1.0, BULL→1.0
   - 建议 BEAR→0.2, NEUTRAL→0.6, BULL→1.0 加大区分度

2. **缩短 filter-confirm-days**
   - 当前=2，建议=1，更快响应市场转换

3. **增加"空仓期"机制**
   - 当BEAR确认且MaxDD已超过阈值时，完全空仓
   - 而非仅减少仓位

4. **Window F 启示**
   - 在真正的长期熊市中，减少仓位确实有效
   - 但需要更激进的position_limit（如0.2而非0.4）
