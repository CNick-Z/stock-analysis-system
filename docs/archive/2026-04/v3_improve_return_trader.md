### 1.2 盈亏比分析（关键问题）

从回测数据计算：

| 指标 | 当前值 | 问题 |
|------|--------|------|
| 胜率 | 61.5% | ✅ 较高 |
| 平均盈利交易 | **~+20%** | ⚠️ 固定20%天花板，被低估 |
| 平均亏损交易 | **~-15%** | ❌ 止损过大 |
| 盈亏比（Reward/Risk） | **1.33** | ❌ 目标应≥2.0 |
| 交易频率 | ~17笔/年 | ✅ 尚可 |

**盈亏比1.33是什么概念？**

$$
\text{期望每笔} = 0.615 \times 20\% - 0.385 \times 15\% = 7.6\%
$$

但这只是"平均单笔期望"，还要乘以：
- 仓位10% → 每笔对账户贡献 = **0.76%**
- 年交易17笔 × 0.76% ≈ **12.9%潜在最大贡献**

理论上一年可以做出12.9%，但实际只有2.65%，说明：
1. 不是所有交易都能满仓10%（市场机会限制）
2. 部分止损是无效的（入场后立即止损，实际损失大于理论值）
3. 止盈被"固定20%"卡住，大量盈利交易没有拿到应有的幅度

**核心结论：平均盈利不够大（被20%天花板压制），是年化低的primary driver。**

## 二、止盈优化：从"固定20%"到"波浪感知型动态止盈"

### 2.1 为什么固定20%是错误的

波浪理论的盈利来源是**W5延伸浪**：
- W1长度：假设10元→15元（+50%）
- W3理论目标：W1涨幅×1.618 = 8元涨幅 → 目标23元（+130%从W1起点）
- W5延伸：更可能走出W3的100%-161.8%

**结论：20%止盈在波浪系统里，相当于W3才刚开始就卖出了。**

### 2.2 分类止盈目标（推荐值）

| 信号类型 | 含义 | 推荐止盈目标 | 理由 |
|---------|------|------------|------|
| **W2_BUY** | W2回调后买入，等W3 | **40%** | W3通常为W1的100-161.8%，给足空间 |
| **W4_BUY** | W4调整后买入，等W5 | **30%** | W5通常为W1的61.8-100%，中等空间 |
| **C_BUY** | 反弹浪，快进快出 | **20%** | C浪是调整，快速结束，合理 |

### 2.3 移动止盈重构（关键！）

**当前问题**：移动止盈只在"超过20%后"生效，且8%回撤太紧。

**重构逻辑**：

```
第一阶段（0% → 止盈目标×50%）：
  - 无移动止盈，让价格自由运行
  - 止损：固定ATR止损

第二阶段（超过止盈目标×50%后）：
  - 激活移动止盈：stop = high × (1 - trailing_pct)
  - trailing_pct：W2=15%, W4=12%, C=10%

第三阶段（超过止盈目标×100%后）：
  - 移动止盈收紧：stop = high × (1 - trailing_pct/2)
  - 锁定更多利润
```

**具体参数**：

| 信号类型 | 止盈目标 | 激活移动止盈（达到%） | trailing_pct | 锁利收紧线 |
|---------|---------|---------------------|-------------|-----------|
| W2_BUY | 40% | 20%后激活 | 15%回撤 | 30%后收紧到8% |
| W4_BUY | 30% | 15%后激活 | 12%回撤 | 22%后收紧到6% |
| C_BUY | 20% | 10%后激活 | 10%回撤 | 15%后收紧到5% |

**代码改动示例**：

```python
# 当前逻辑（有缺陷）
if next_open >= profit_price:  # 20%就止盈
    return True, f"TAKE_PROFIT @{next_open:.2f}"

trailing_stop = high_since_entry * 0.92  # 只有20%+才生效，8%太紧

# 改进逻辑
PROFIT_TARGET = {'W2_BUY': 0.40, 'W4_BUY': 0.30, 'C_BUY': 0.20}
TRAILING_ACTIVATE = {'W2_BUY': 0.20, 'W4_BUY': 0.15, 'C_BUY': 0.10}
TRAILING_PCT = {'W2_BUY': 0.15, 'W4_BUY': 0.12, 'C_BUY': 0.10}

target = PROFIT_TARGET.get(signal_type, 0.20)
activate = TRAILING_ACTIVATE.get(signal_type, 0.10)
trail_pct = TRAILING_PCT.get(signal_type, 0.10)

profit_price = entry_price * (1 + target)

# 第一阶段：未达到激活线，正常持有多
if next_open >= profit_price:
    return True, f"TAKE_PROFIT @{next_open:.2f}"

# 第二阶段：激活移动止盈
high_since_entry = pos.get('high_since_entry', 0)
entry_to_high_pct = (high_since_entry / entry_price - 1) if entry_price > 0 else 0

if high_since_entry > 0 and entry_to_high_pct >= activate:
    # 30%后收紧trailing
   收紧线 = target * 0.75
    actual_trail = trail_pct / 2 if entry_to_high_pct >= 收紧线 else trail_pct
    trailing_stop = high_since_entry * (1 - actual_trail)
    if next_open <= trailing_stop:
        return True, f"TRAILING_STOP({high_since_entry:.2f}→{next_open:.2f})"
```

## 三、止损优化：从"固定5%"到"ATR动态止损"

### 3.1 当前固定止损的问题

- 5%止损对高波动股票（经常±10%/日）完全无效
- 对低波动股票（±2%/日）又太宽松，白白浪费入场点位

从回测数据看，止损出场的平均损失是-15%，远超5%的设计值。说明：
1. 5%止损根本守不住
2. 高波动股票在入场后迅速触及5%止损，但价格很快反弹（被洗出）
3. 而那些最终触发止损的，都是真趋势反转，-15%起

### 3.2 ATR止损推荐方案

**公式**：
$$
\text{止损价} = \text{入场价} - k \times ATR_{20}
$$
$$
\text{其中 } k = 1.5 \text{（保守）} \sim 2.0 \text{（激进）}
$$

**推荐k=1.8**（在保护性和不被洗出之间取得平衡）

**参数表**：

| 股票波动率 | ATR/价格比 | k=1.8止损距离 | 实际止损% |
|-----------|-----------|-------------|---------|
| 低波动（如大盘蓝筹） | 1.5% | ATR×1.8 = 2.7% | 2.7% |
| 中波动（大多数） | 3% | ATR×1.8 = 5.4% | 5.4% |
| 高波动（题材股） | 6% | ATR×1.8 = 10.8% | 10.8% |

**优势**：止损距离自动适应个股波动率，低波动股止损紧，高波动股止损宽。

### 3.3 双重止损机制

```python
# 止损1：ATR动态止损（主要）
atr = row_dict.get('atr20', 0)
if atr > 0:
    atr_stop = entry_price - 1.8 * atr
else:
    atr_stop = entry_price * 0.95  # fallback到5%

# 止损2：保本线（防止滑点导致的小亏损累积）
breakeven = entry_price * 1.01  # +1%保本

# 最终止损：取较近者
final_stop = max(atr_stop, breakeven)  # 保本线和ATR止损的较优者
```

### 3.4 预期效果

| 场景 | 当前 | 优化后 |
|------|------|-------|
| 高波动股票止损 | -15%~-35% | -8%~-12% |
| 低波动股票止损 | -5%（合理） | -3%（更精准） |
| 被洗出后反弹 | 频繁发生 | 减少（ATR过滤假突破） |

## 四、仓位优化：按信号置信度分配

### 4.1 当前问题

当前所有信号统一10%仓位。但不同信号类型的风险收益比完全不同：

| 信号类型 | 风险 | 潜在收益 | 盈亏比 | 建议仓位 |
|---------|------|---------|--------|---------|
| W2_BUY | 中等（W2可能失败） | 高（W3通常>50%） | ~3:1 | **15%** |
| W4_BUY | 中等偏高（W4复杂） | 中等（W5通常30-50%） | ~2:1 | **10%** |
| C_BUY | 高（C浪不确定） | 低（反弹20%） | ~1:1 | **5%** |

### 4.2 分层仓位方案

```python
POSITION_BY_SIGNAL = {
    'W2_BUY': 0.15,       # 最高置信，趋势启动
    'W4_BUY': 0.10,       # 中等置信，调整浪
    'C_BUY': 0.05,        # 低置信，反弹浪
    'W4_BUY_ALERT': 0.05, # 预警信号，轻仓试探
}
```

**为什么不更高？**

分散持仓10只是为了防个股黑天鹅。10只×15% = 150%仓位，已经比较激进。

如果想提高到20%单股，需要先证明ATR止损有效性。

### 4.3 动态仓位调整（进阶）

```python
def calc_position_size(entry_price, stop_loss, wave_confidence):
    """根据止损距离和波浪置信度动态计算仓位"""
    risk_ratio = 0.02  # 每笔最多亏账户2%
    
    stop_distance = entry_price - stop_loss
    if stop_distance <= 0:
        return 0
    
    # 理论股数
    shares_by_risk = (account_balance * risk_ratio) / stop_distance
    
    # 按置信度调整
    confidence_multiplier = {'W2_BUY': 1.2, 'W4_BUY': 1.0, 'C_BUY': 0.6}
    mult = confidence_multiplier.get(signal_type, 0.8)
    
    final_shares = int(shares_by_risk * mult // 100 * 100)
    return max(final_shares, 100)
```

## 五、持仓周期优化

### 5.1 当前问题

| 信号 | 当前时间止损 | 问题 |
|------|------------|------|
| W2_BUY | 无（合理） | ✅ 可以走出完整W5 |
| W4_BUY | **25天** | ❌ 25天内W5可能走不完 |
| C_BUY | **15天** | ⚠️ 合理，但止盈20%太紧 |

**典型问题**：一个健康的W4→W5上涨，需要时间形成笔。
- W4调整时间：通常2-4周（合理）
- W5上涨时间：通常3-8周
- 总周期：5-12周

W4_BUY限25天，意味着**W5还没走完就被迫出场**。

### 5.2 持仓周期推荐

| 信号类型 | 当前 | 推荐 | 理由 |
|---------|------|------|------|
| W2_BUY | 无限制 | **60天**（约3个月） | W3/W5需要时间，可以更长 |
| W4_BUY | 25天 | **40天** | W5通常需要3-6周 |
| C_BUY | 15天 | **20天** | 反弹浪，快速结束 |

### 5.3 时间止损≠时间强制平仓

关键区别：
- **错误做法**：满25天，无论盈亏都平仓
- **正确做法**：满40天时，**重新评估** wave_state：
  - 如果 wave_state 仍然是 'w4_in_progress' 或 wave_trend='up' → 继续持有
  - 如果 wave_trend='down' 且持仓亏损 → 保本出
  - 如果 wave_trend='down' 且持仓盈利 → 止盈出

```python
# 改进的时间止损逻辑
if days_held >= max_days:
    wave_state = row_dict.get('wave_state', '')
    wave_trend = row_dict.get('wave_trend', '')
    entry_price = pos.get('avg_cost', 0)
    current_price = next_open
    
    if wave_trend == 'up':
        return False, "TIME_BUT_TREND_UP"  # 趋势还在，继续持有
    elif current_price > entry_price:
        return True, f"TIME_EXIT_PROFIT({days_held}天)"
    else:
        return True, f"TIME_EXIT_BREAKEVEN({days_held}天)"
```

## 八、关键原则总结

1. **让盈利交易跑更远**：40%止盈目标，让W3/W5走完
2. **用移动止盈保护利润**：不是"到目标就卖"，而是"越高越不能跌"
3. **止损要自适应**：固定止损在高波动市场是个笑话，ATR才是答案
4. **信号分强弱，仓位有差异**：不是所有信号都配10%仓位
5. **持仓周期是朋友**：W4→W5需要时间，25天太急
